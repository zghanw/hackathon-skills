# Contributing

The bar for adding anything: **would a team use it during an actual
hackathon weekend?** This repo stays at 3 skills on purpose: the value of a
lean install is the whole pitch.

Most wanted:

- **Scenario rows** for [SCENARIOS.md](SCENARIOS.md): a situation you hit at
  a real hackathon and the skill that solved it (with the install command
  verified against that repo's current README, not a blog post).
- **Judge questions** for [references/judge-questions.md](references/judge-questions.md):
  questions you were actually asked, with what a strong answer contained.
- **Template improvements**: fields you added to `HACKATHON.md`/`PITCH.md`
  mid-hackathon because the template missed them.
- **War stories as examples**: a filled-in (anonymized) `examples/` pair
  from a real event beats our fictional one.

New skills need a stronger case: name the phase the existing three plus the
companions in SCENARIOS.md don't cover, and why a guide row can't cover it.

Open a PR. CI validates the manifests and skill frontmatter automatically.
